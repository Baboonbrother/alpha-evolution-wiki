# 實驗 008：世界模型 Alpha 第一輪——世界層訊號 dev 全過、validation 全崩

這是第一次**用世界模型找 Alpha**的正式實驗：訊號不再是個股的性質（價格路徑、個股營收），而是**世界的性質**——產業層需求狀態。四個候選全部從世界鏈唯一有真資料的那一跳（[[graph-native|產業營收世界狀態]]）的不同面向長出，其中一個直接編碼了 king2 最大殘差的機制（陽明：產業轉上、個股營收還沒印出來）。

一句話結果：**dev（2014-2021）四個候選全數強顯著（t=5.8~16.5），prereg 凍結後 validation（2022-2024）全數歸零、零挑戰者**。逐年拆解顯示這不是單一事件假象，而是一個真的世界模型發現：**世界層訊號本身有 regime 相依性**。

## 假說：報酬的橫斷面資訊藏在「世界層」而非只在「個股層」

king2 的分數全部由個股性質組成（個股營收排名、指紋、籌碼、品質）。殘差分析（[[exp-007-residual-belief|exp-007]]）指出它系統性漏掉「產業需求轉上、個股數字還沒跟上」的股票——最重案例陽明 2020-12：航運業營收 YoY 從 −13.6% 轉上、陽明自己的 rev_rank 只有 87 名 → 落選 → 超額 +134%。

若世界模型的傳導鏈「產業需求 → 產業營收 → 個股重定價」為真，那**產業層的營收狀態應該對橫斷面報酬有預測力**。四個候選各測傳導的一個面向：

| 候選 | 定義 | 測世界狀態的哪個面向 |
|---|---|---|
| `W_LEVEL` | 產業營收 YoY 中位（B-RES-001 的軸，日頻化） | 水位 |
| `W_ACCEL` | 產業營收 YoY 三月變化 | **加速度**（超級週期簽名） |
| `W_BREADTH` | 產業內 YoY>0 成員比例 | **擴散度** |
| `W_LAGGARD` | RankPct(產業加速度)×(1−RankPct(個股YoY)) | **陽明機制**：產業轉上、個股尚未反映的領先期 |

面板構造 PIT by construction：月營收 index＝公告截止日、as-of ffill 只用過去值；產業成員身分用當前快照（輕微 look-ahead，與 B-RES-001 同 caveat，誠實列出）。

## 紀律：prereg 凍結先於看 validation，Bonferroni 隨測次抬門檻

流程三段：**dev 篩選**（研究段可自由看）→ **prereg 凍結**（存活候選＋六門協議 sha256 入 `prereg_freeze`，時戳早於任何 validation 統計）→ **validation 確認**（G2 樣本外 IC 同號顯著／G3 淨成本投組 Sharpe≥0.5 且多頭腿為正／G5 對 king2 鏡像正交／動能閘）。搜尋帳 family=`WORLD_STATE`，八次檢定門檻從 2.0 一路抬到 4.1——**防「測多了挑幸運者」的機制真的在漲價**。vault（2025+）未碰。

## 結果：dev 與 validation 的斷裂乾淨得像教科書

| 候選 | dev IC（naive t → **NW t**） | val IC（NW t） | 淨成本多空 Sharpe | 裁決 |
|---|---|---|---|---|
| W_LEVEL | 0.0228（10.5 → **2.95**，真過） | 0.0006 (0.04) | −0.46 | BLOCKED@G2 |
| W_ACCEL | 0.0102（5.8 → **1.81**，⚠假過） | 0.0033 (0.23) | −0.15 | BLOCKED@G2 |
| **W_BREADTH** | 0.0354（16.5 → **4.63**，真過） | −0.0029 (−0.22) | 0.08 | BLOCKED@G2 |
| W_LAGGARD | −0.0154（−10.0 → **−2.82**，⚠假過） | −0.0030 (−0.29) | 0.11 | BLOCKED@G2 |

**先講對抗稽核揪出的實錘（NW t 那一欄的由來）**：20 日前瞻窗每日重疊使 IC 序列自相關達 0.942，naive t **系統性高估 3.5~4 倍**。Newey-West（lag=20）修正後，「四候選全數強顯著」縮水成「**兩條真過**（W_BREADTH 4.63、W_LEVEL 2.95），兩條是重疊灌水的假 dev pass」。閘已改用 `t_nw`（`harness.nw_tstat`），歷史結果檔以 `nw_correction` 節事後誠實註記（不改寫原欄位）。

**逐年拆解是本實驗最有價值的產出**（以最強的 W_BREADTH 為例）：

```
2014 +0.037  2015 +0.045  2016 +0.044  2017 +0.045
2018 +0.015  2019 +0.032  2020 +0.063
─── 2021 +0.003 ≈ 0：斷點已在 dev 尾巴出現 ───
2022 −0.003  2023 +0.018  2024 −0.023（NW t≈−4.2，強翻負）
```

dev 期**年年非負**（平均 ≈+3.5%，2020 超級週期只是最強一年）——但精確的敘事是：**斷點在 2021 就出現了**（仍在 dev 內、IC 已歸零），之後是漸進震盪死亡而非 2022 斷崖（2023 短暫轉正、2024 強烈翻負）。「一條 2014-2020 真實存在、之後漸進死亡並於 2024 翻負的世界層訊號」是比「2022 起 regime 斷裂」更誠實的描述。

另一個誠實的意外：`W_LAGGARD`（陽明機制）的 dev IC 方向是**負的**（NW 修正後 −2.82、未過該次門檻 2.9）——「產業轉上但個股還沒跟上」的股票在橫斷面均值上**沒有**正向優勢；陽明式的大贏是右尾事件，不是均值行為。機制敘事在個案上成立、在橫斷面均值上找不到支持，這正是「個案→假說→橫斷面檢定」這條鏈存在的理由。

## 裁決與這對世界模型的意義

**裁決：四候選全 BLOCKED、零挑戰者（誠實負結果）。** 但這輪買到的知識比一條弱 Alpha 值錢：

1. **世界層訊號是真的存在過的**——W_BREADTH 經 NW 修正後 t≈4.6、2014-2020 年年為正，不是雜訊；「報酬資訊藏在世界層」的假說方向沒被否證。
2. **它 regime 相依（描述性）**——訊號在 2021 歸零、之後漸進震盪死亡、2024 翻負。這回饋給世界模型一個具體的結構需求：**世界層訊號需要更高一層的世界狀態（宏觀 regime）來條件化**——「什麼時候該聽產業營收說話」本身是一個世界狀態問題。2024 的強翻負依[[blockers|不可預測宣稱閘]]先入殘差清單：「regime 相依」目前是標籤不是機制，升格成解釋前要有機制檢定（例如翻負是否＝訊號被定價／擁擠反轉）。
3. **紀律全程正確運作、且測量層也被修了**：prereg 先於樣本外、Bonferroni 抬門檻、成本閘、對 king2 正交——dev naive t=16.5 的訊號被 validation 擋下；而稽核連「dev 很強」這句話本身都砍了一刀（naive t 灌水 3.5~4 倍），閘從此改用 NW t。**這台機器又一次拒絕了自己的漂亮數字——包括拒絕自己誇大的顯著性。**

## 對抗查證：MOSTLY_TRUSTWORTHY_CAVEATS（骨幹成立＋一條實錘灌水已修）

三個獨立稽核鏡頭實跑攻擊本輪結果：

- **假陰性狩獵（val 崩壞是不是資料假象）→ SOLID**：面板覆蓋 dev=val（0.925）、月營收無斷更、ffill 陳舊度兩期分布重疊（中位 14 天）；用獨立實作（scipy 自建遮罩）逐位重現 dev 16.6／val −0.86——**崩壞是真的，不是計算器單邊 bug**。
- **dev 灌水狩獵 → SOLID＋一條實錘**：生存者偏誤排除（finlab 面板保留 376 檔下市股與 497 條死營收序列，排除方向反而不利 dev）；產業快照踢除通道極小（25/1762）且方向壓低非灌高；**實錘＝重疊窗 naive t 高估 3.5~4 倍**（上表 NW 欄，已修）。
- **紀律完整性 → SOLID**：prereg 時戳嚴格在 dev 四測後、val 四測前；WORLD_STATE 家族恰 8 筆、門檻 2.0→4.1 遞增吻合；vault 零觸碰；**方向誠實在結構上封死**——sgn 由 dev 定，實測 val 用反方向淨 Sharpe −0.99（報告方向 +0.08），且 G2 同號要求使任何選邊都救不活。揪出的記帳小洞（validation 查詢未寫 holdout_query_log）已修。

## 誠實邊界（不得省略）

- **「regime 相依」目前是描述不是已檢定的假說。** 「用宏觀 regime 條件化世界層訊號」是下一個該 prereg 的假說——但注意三條規則（稽核開的）：regime 變數必須 ex-ante 可定義（只用 dev 期選定、凍結後才碰 val）；WORLD_STATE 家族已測 8 次、下一筆門檻 t=4.4，要先想清楚值不值得燒；不得為了降門檻另開家族。
- **產業成員身分當前快照**＝輕微 look-ahead（稽核量化：踢除通道僅 25/1762 檔、方向壓低非灌高）；生存者偏誤已排除（finlab 面板保留下市股歷史）。
- **naive t 灌水已實錘並修正**：重疊窗使 naive t 高估 3.5~4 倍；本模組閘已改用 `t_nw`。**其他頁面歷史 naive t（如開放探索的 low_vol t=15）同受此高估影響**——讀時要打 3.5~4 倍折扣（那些候選本就被成本閘擋下，結論不變）；整個評分器（`evaluate`/`judge`）換 NW 屬 evaluator_version 升版，列為待辦。
- **validation 是第一保留集**：本輪用掉了它對這四個候選的檢定力；同家族後續假說門檻更高（現已到 4.4）。confirmed 只能等 LIVE_FORWARD。**四候選一條都不得餵 paper-account**——W_BREADTH 的 dev 期強度只能當 regime 條件化研究的種子問題，不是策略。
